# Repository Topology Baseline

## Status

Stable Baseline

---

## Purpose

This document defines the stable organizational roles of the primary repositories within the Research Program.

The purpose is not to document the current directory tree.

Instead, this document provides the organizational principles used for long-term repository placement and maintenance.

It serves as a stable reference for repository placement decisions across the Research Program.

---

# Organizational Principle

Repository placement is determined according to the primary organizational role of each research asset rather than temporary convenience.

Each completed asset should be placed in the repository whose long-term purpose most closely matches the asset's organizational function.

# Placement Principle

Repository placement should prioritize:

1. Organizational role
2. Long-term maintainability
3. Repository consistency
4. Relationship to existing assets

Local convenience should not override repository consistency.

---

# Repository Relationship

The Hub Repository describes the architecture of the Research Program.

The Dialogue Repository stores the research assets generated within that architecture.

Together, they separate organizational knowledge from research production while maintaining clear long-term structure.

---

# Intended Use

This document serves as the baseline reference for repository placement.

Individual placement decisions should remain consistent with the organizational principles defined here.

Detailed placement recommendations may evolve over time, but the primary repository roles should remain stable whenever possible.
